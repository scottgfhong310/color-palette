# 圖庫肖像：關係與時間（Collection Portrait）

> 這份文件回答兩件事：**怎麼理解「關係與時間」**（尤其**時間**在這裡是怎麼呈現的）、
> **透過哪些資料**來描述它。
> 這是色彩肖像從「一張圖」擴到「一批圖」的一步。相關：色彩肖像 [COLOR-PORTRAIT.md](COLOR-PORTRAIT.md)｜
> 可查詢標籤 §「metadata」｜五類色型 [COLOR-TYPES.md](COLOR-TYPES.md)。

## 1. 怎麼理解「關係與時間」

單張的色彩肖像回答「**這一張**是什麼氣質」。當你有**一整櫃圖**，還有兩個新問題：

- **關係**：這**一批**圖合起來是什麼樣子？（整體偏暖？多是大地色？和諧以互補居多？）
- **時間**：這批圖**隨時間怎麼變**？（越近期越暖？越鮮？）

`collectionPortrait()` 把它算成一行摘要，顯示在 gallery 頂端：

> 「83 張 · 整體偏暖 · 單色為主 · **近期偏暖**」

### 「關係」＝描述一個「集合」，而且尊重篩選

「關係」有兩層：

1. **集合的整體肖像**：把這批圖的冷暖、最常見的配色原型與和諧，濃縮成幾個詞。
2. **相對/子集**：摘要描述的是**目前可見的那一批**——所以**篩選會改變它**。
   篩「互補」→「9 張 · 整體偏暖 · 多為大地色調 · **互補為主**」。
   於是它不只是「整櫃概觀」，也是「**任意你圈出來的一組圖**的概觀」。

### 「時間」在這裡是「趨勢方向」，不是時間軸

**這裡沒有時間軸、沒有折線圖。** 時間被收斂成**一個方向判斷**——這批圖正在「往哪個方向漂」。

做法：把這批圖依**時間排序**，切成**較舊的一半**與**較新的一半**，比較兩半的平均冷暖與彩度：

```
   依 mtime 排序：  舊 ────────────────► 新
                   [ 較舊的一半 ]   [ 較新的一半 ]
                        │                │
                     平均暖冷 / 彩度   平均暖冷 / 彩度
                        └──── 差 ─────┘  →  近期偏暖 / 偏冷 / 偏鮮 / 偏濁
```

所以「時間」的呈現＝**一句「近期偏X」**：新的一半比舊的一半更暖，就是「近期偏暖」。
它回答的是「**這櫃圖的用色正往哪裡走**」，而不是「每個時間點各是什麼」。
（≥6 張才判趨勢；差距不夠明顯就不說。）

> 為什麼這樣呈現：摘要是**一行字**，不是儀表板。時間濃縮成「方向」最省、也最好懂；
> 兩半相比（而非逐張時間序）對少量、不規則上傳的圖庫最穩健。

## 2. 透過哪些資料

摘要的輸入是**目前可見的每一張圖**（尊重篩選）的三樣東西：

| 每張圖的資料 | 來源 | 算出什麼 |
|---|---|---|
| **`colors`** | 該圖的 alias 色票 | 經 `metrics()` → **暖度**（暖−冷面積）、**彩度**（平均飽和） |
| **`tags`** | 分析時**落地的色彩肖像標籤**（`archetype:…`／`harmony:…`） | 該圖的**配色原型**與**和諧方案** |
| **`mtime`** | 檔案時間（實務上＝上傳時間） | **時間排序**用 |

**為什麼原型/和諧用 tags 而不重算**：篩選也是用這批落地 tags 比對的。若摘要另外從 alias 重算，
會出現「篩了互補、摘要卻說單色」的**打架**。統一用 tags → 摘要與篩選**同源、一致**。

### 聚合成一行

```
可見的一批圖（尊重篩選）
  每張 → colors ─metrics()─► warmth, chroma
        → tags ──────────► archetype, harmony
        → mtime ─────────► 時間排序
聚合：
  warmth 平均              → 整體偏暖 / 偏冷 / 中性
  archetype/harmony 眾數   → 「多為 X」/「X 為主」（需覆蓋 ≥30% 才說）
  舊半 vs 新半 的 warmth/chroma 差 → 趨勢「近期偏 X」（取差距最大的一項）
```

`collectionPhrase()` 把這些湊成一行、以 ` · ` 分隔；措辭在 `collection.*` locale。純函式、不碰 DOM。

## 一個實例

83 張的圖庫，未篩選：

- **關係**：多數圖暖度為正 → **整體偏暖**；和諧的眾數是單色且覆蓋 ≥30% → **單色為主**；
  沒有單一原型覆蓋 ≥30% → 原型這格不說。
- **時間**：依 mtime 分兩半，較新一半的平均暖度高於較舊一半 → **近期偏暖**。

合起來：**「83 張 · 整體偏暖 · 單色為主 · 近期偏暖」**。

篩「互補」後，這批變成 9 張，摘要跟著重算：**「9 張 · 整體偏暖 · 多為大地色調 · 互補為主」**。

## 誠實的邊界

- **`mtime` 是檔案時間（上傳）**，不是畫作的創作年代；「時間趨勢」講的是「你**收進圖庫**的順序」。
- **趨勢是粗略的兩半相比**，不是精細時間序；只給方向，不給曲線。
- **「相對某個外部畫派/攝影師」尚未做**——目前只相對「本圖庫／本篩選子集」。要做需要一個外部參考集。
